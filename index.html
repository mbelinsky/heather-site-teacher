PHASE 01 / SETUP

BEFORE YOU
TYPE ANYTHING

Claude works best when it knows who you are. This phase is about giving it context before the first prompt lands.

Create Your Project

Claude Projects keep your whole site in one conversation. Every decision, draft, and file in one place you can return to.

→ claude.ai → New Project → name it after your site

Write Your Preferences

Your preferences doc is the most important thing you'll write. It tells Claude your aesthetic, your audience, your voice. Before any design work: fill out the AITO framework below.

Connect Your Tools

Add the Cloudflare MCP connector. This lets Claude deploy your site directly from the conversation.

→ Settings → Connectors → Cloudflare Developer Platform
AUDIENCE Who visits. What they should feel. What they should do.
IDENTITY Your passions. Your personality in one sentence.
TASTE 3 sites you love. 3 words that are NOT you.
OUTPUT How Claude should explain decisions. Format rules.

PHASE 02 / COPY

WORDS FIRST.
DESIGN SECOND.

Every great site starts with a brief. Three questions that force you to think before you type.

01
WHAT is the one thing someone should understand after visiting?
02
HOW should they feel when they get there?
03
WHAT should they do next?
"I'm writing copy for my personal website. Help me answer: 1. The ONE thing I want someone to understand after visiting. 2. How they should feel. 3. What they should do next. About me: [your identity section from Phase 01] Sections needed: [hero, about, projects, contact] Don't write copy yet. Just help me nail the brief."
"Now write copy for each section. For each: headline A (bold/direct) + headline B (subtle/evocative), body copy max 3 sentences, microcopy for buttons and captions. Write like a real person, not a brochure."
03
PHASE 03 / DESIGN

COMMIT TO AN
AESTHETIC

Vague direction produces generic results. Name your aesthetic before Claude touches a line of CSS.

COLOR

1 primary · 1 accent · neutral bg · dark text

TYPOGRAPHY

1 display font + 1 body font. Never defaults.

SPACING

Pick 8px as your base. Everything is multiples.

COMPONENTS

Define button, card, nav once. Reuse everywhere.

"Based on these references: [URLs] Extract: color palette with hex values and mood of each, font pairing from Google Fonts with reasoning, spatial direction (tight/airy, symmetric/not), one signature move that makes this site feel like mine. Output a design brief I can paste into the next prompt."
"Commit to one aesthetic direction. Name it. Describe it in 2-3 sentences — what it IS and what it deliberately is NOT. Then give me: --color-bg, --color-text, --color-accent, --color-secondary, --font-display, --font-body, --spacing-unit"
04
PHASE 04 / BUILD

THREE PROMPTS.
ONE SITE.

The build phase runs in Claude Code, not the chat interface. Claude Code has terminal access — it can create files, run servers, and show you a live preview as it builds.

Start with the scaffold prompt. Get the bones right before touching any section content. Polish last, not during.

"Build the HTML scaffold for my personal website. Design direction: [paste named aesthetic + CSS variables] Copy: [paste all copy from Phase 02] Sections: hero, about, work, contact Use the frontend-design approach — visually distinctive, not generic. Single index.html, Tailwind via CDN for utilities, custom CSS for personality. Before coding: tell me your spatial composition plan and what the unforgettable thing about this design is."
"Build the [SECTION NAME] section. Make it consistent with [previous section]. Copy: [paste copy for this section] Interactions: [describe or say 'your call']"
"Do a polish pass. Check: - Typography consistency - 8px spacing rhythm - Animation cohesion - Mobile under 375px - One thing that feels underdeveloped — fix it. Show me the updated full file."

PHASE 05 / DEPLOY

YOUR SITE. LIVE. TODAY.

GITHUB

Where your code lives. Every save is a version you can return to. Every push triggers a deploy.

  • Step 1 Create account at github.com
  • Step 2 Install GitHub Desktop (not CLI — not yet)
  • Step 3 New repo → name it your project name → Public
  • Step 4 Drag index.html into GitHub Desktop
  • Step 5 Write a commit message → Push to main
"Walk me through setting up GitHub Desktop for the first time on [Mac/Windows]. Steps only."

CLOUDFLARE

Free hosting. Global CDN. Auto-deploys when you push. Zero config for static HTML.

  • Step 1 Create free account at cloudflare.com
  • Step 2 Workers & Pages → Create → Pages → Connect to Git
  • Step 3 Authorize GitHub → select your repo
  • Step 4 Build settings: framework = None, command = blank, output directory = blank
  • Step 5 Deploy → you get [yoursite].pages.dev
"I have a static index.html in a GitHub repo called [name]. Connect it to Cloudflare Pages. Tell me exactly what to click, in order."

YOUR DOMAIN

A real URL that's yours. ~$10/year. Buy through Cloudflare Registrar — DNS auto-configures.

  • Step 1 Cloudflare → Domain Registration → search your name
  • Step 2 Purchase (at-cost, no markup)
  • Step 3 Pages → your project → Custom Domains → add domain
  • Step 4 Cloudflare auto-sets DNS → live in minutes
"I want to buy [domain.com] and connect it to my Cloudflare Pages site. Exact steps, what to click."
